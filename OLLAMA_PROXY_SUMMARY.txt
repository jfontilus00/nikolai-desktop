OBJECTIVE
================================================================================

Implement Ollama proxy system in Rust backend to bypass Tauri MSI/EXE 
fetch/streaming restrictions while maintaining browser dev mode functionality.

Problem: Tauri packaged apps (MSI/EXE) have fetch restrictions that prevent
direct Ollama API calls. Solution: Route all Ollama requests through Rust 
backend using reqwest.

FILES MODIFIED/CREATED
================================================================================

1. src-tauri/src/ollama_proxy.rs (NEW - 5671 bytes)
   ──────────────────────────────────────────────
   • Rust module for Ollama proxy
   • Uses CancellationToken for stream cancellation
   • Event-based streaming (ollama://token, ollama://done, ollama://error)
   • Functions:
     - ollama_tags(base_url) → List models
     - ollama_chat_once(base_url, body) → Non-streaming chat
     - ollama_chat_stream(window, id, base_url, body) → Streaming chat
     - ollama_chat_abort(id) → Cancel ongoing stream
   • Handles both /api/chat and /api/generate response formats
   • Proper error handling and resource cleanup

ARCHITECTURE

FRONTEND (TypeScript)
  │
  ├── isTauri() detection
  │   ├── true → invoke("ollama_*", { ... })  [MSI/EXE mode]

  │   ├── Event emission to frontend

  └── main.rs integration
